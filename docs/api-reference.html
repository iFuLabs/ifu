API Reference

The iFu Labs Comply API is a REST API that gives you programmatic access to compliance controls, scan results, evidence, vendors, and cost optimisation data. All responses are JSON.

Base URL https://api.ifu-labs.io Production
Base URL http://localhost:3000 Local development
Authentication

All API endpoints require a valid Auth0 JWT token in the Authorization header. Tokens are issued by Auth0 after the user completes the login flow.

🔒 All requests must include: Authorization: Bearer <token>
Example request
curl https://api.ifu-labs.io/api/v1/auth/me \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVC..."
Tokens expire after 24 hours. Your frontend Auth0 SDK will handle token refresh automatically. For server-to-server calls, use the Auth0 Management API to generate a machine-to-machine token.

All data is scoped to the authenticated user's organisation. You cannot access another organisation's data regardless of what IDs you provide.

Errors

The API uses standard HTTP status codes. Error responses always include a JSON body with error and message fields.

StatusMeaningCommon cause
200OKRequest succeeded
201CreatedResource created successfully
204No ContentDELETE succeeded
400Bad RequestValidation error or missing required field
401UnauthorizedMissing or invalid JWT token
403ForbiddenValid token but insufficient role (e.g. member trying admin action)
404Not FoundResource doesn't exist or belongs to another org
409ConflictResource already exists (e.g. already onboarded)
429Rate Limited100 requests per minute per IP
503Service UnavailableAI or external service temporarily unavailable
Error response shape
{
  "error": "Not Found",
  "message": "Integration not found"
}

Integrations

Connect cloud accounts to enable automated compliance scanning. iFu Labs uses read-only access only — we never write to your infrastructure.

GET /api/v1/integrations List integrations ▼

Returns all integrations for this org. Raw credentials are never returned — only metadata and status.

Response item shape
{
  "id": "uuid",
  "type": "aws",
  "status": "connected",
  "metadata": { "accountId": "123456789012", "alias": "acme-prod" },
  "lastSyncAt": "2026-04-12T02:00:00.000Z",
  "lastError": null
}
POST /api/v1/integrations/aws Connect AWS account ▼

Connects an AWS account via cross-account IAM role. iFu Labs validates the role can be assumed before saving.

⚠ The IAM role must have the SecurityAudit managed policy attached and a trust policy allowing assumption from the iFu Labs AWS account with the provided external ID.
FieldTypeRequiredDescription
roleArnstringrequiredARN of the read-only IAM role. Format: arn:aws:iam::ACCOUNT:role/ROLE_NAME
externalIdstringrequiredExternal ID shown to the user during setup. Prevents confused deputy attacks.
Request body
{
  "roleArn": "arn:aws:iam::123456789012:role/iFuLabsComplyRole",
  "externalId": "ifu-labs-abc123xyz"
}
201Integration created, initial scan queued
400Role cannot be assumed — check trust policy
POST /api/v1/integrations/github Connect GitHub org ▼

Connects a GitHub organisation via GitHub App installation. The installation ID is found in the URL after installing the app.

FieldTypeRequiredDescription
installationIdintegerrequiredGitHub App installation ID from github.com/settings/installations/ID
Request body
{ "installationId": 12345678 }
201Integration created, initial scan queued
400Installation not found
POST /api/v1/integrations/:id/sync Trigger manual scan ▼

Queues an immediate compliance scan for the integration. Returns a job ID for tracking.

Response
{ "message": "Scan queued", "jobId": "123" }
200Scan queued
400Integration not connected
DELETE /api/v1/integrations/:id Disconnect ▼

Disconnects an integration and removes stored credentials. Scan history and control results are preserved. Requires admin role.

204Disconnected
Evidence

Evidence library including automated scan results and manual uploads. Export full audit packs as PDF.

GET /api/v1/evidence/export/pdf Export audit PDF ▼

Generates and streams a compliance evidence pack as PDF. The response is a binary PDF file.

Query paramTypeDefaultDescription
frameworkstringsoc2soc2 · iso27001 · gdpr · hipaa · pci_dss
The PDF includes: cover page, table of contents, executive summary, per-category control sections with evidence, remediation guidance for failures, and scan history.
200PDF binary — Content-Type: application/pdf
GET /api/v1/evidence List evidence ▼

Returns all evidence split into automated (from scans) and manual (uploaded) arrays.

200{automated: [], manual: [], total: N}

Adds a manual evidence item — e.g. a policy document, vendor contract, or training record.

Generates a structured AI explanation for a failing control. Results cached for 24 hours per org per control.

GET /api/v1/ai/explain/:controlId/stream Stream explanation (SSE) ▼

Same as POST /ai/explain but streams token-by-token via Server-Sent Events. Use for real-time UI rendering.

Event stream format

Vendors

Third-party vendor risk tracking. Tracks SOC 2 and ISO 27001 cert expiry with 60-day alerts.

All 5 vendor endpoints (GET list, GET by ID, POST, PATCH, DELETE) follow standard REST conventions. See the Postman collection for full request/response examples.
Cert status values
"soc2Status": "valid"      // cert expires > 60 days away
"soc2Status": "expiring"   // cert expires within 60 days
"soc2Status": "expired"    // cert has expired
"soc2Status": "none"       // no cert date recorded

Organisation settings and team member management. Role hierarchy: owner > admin > member.

RoleCan do
ownerEverything — including changing member roles and billing
adminConnect/disconnect integrations, manage vendors and evidence
memberRead everything, add notes, export PDFs
